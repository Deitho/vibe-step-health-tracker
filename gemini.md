# Personal Health Tracker - Project Guidelines

## Project Overview
- **Framework:** Next.js
- **Architecture:** Single Page Web Application (SPA)
- **Deployment:** Vercel (to be deployed once finished)
- **Integrations:** GitHub and Vercel MCP server connections (to be configured later)

## Goals
- Build a responsive, user-friendly personal health tracking application.
- Maintain a clean, maintainable, and modern codebase.
- Prepare the application for seamless deployment to Vercel.

*Detailed Standard Operating Procedure (SOP) document will be provided and should be referenced for specific implementation details.*

## Operating Principles
1. **Component-Driven Development:** Build reusable, modular UI components.
2. **State Management:** Use modern React state management suitable for an SPA.
3. **Styling:** Use a structured styling approach (e.g., Tailwind CSS or Vanilla CSS as preferred by the project setup).
4. **Iterative Development:** Implement features according to the provided SOPs, ensuring each step is tested and verified.
